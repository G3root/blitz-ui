# Textarea

The `Textarea` component allows you to easily create multi-line text inputs.

### Import

```js
import { Textarea } from '@blitz-ui/react';
```

### Usage

<ComponentContainer>
  <Textarea placeholder="Here is a sample placeholder" />
</ComponentContainer>

```jsx
<Textarea placeholder="Here is a sample placeholder" />
```

### Textarea sizes

Use the `size` prop to change the size of the Textarea's text.
You can set the value to `xs`, `sm`, `md` or `lg`.

<ComponentContainer>
  <Textarea placeholder="extra small size" size="xs" />
  <Textarea placeholder="small size" size="sm" />
  <Textarea placeholder="medium size" size="md" />
  <Textarea placeholder="large size" size="lg" />
</ComponentContainer>

```jsx
<div className="flex items-center gap-2">
  <Textarea placeholder="extra small size" size="xs" />
  <Textarea placeholder="small size" size="sm" />
  <Textarea placeholder="medium size" size="md" />
  <Textarea placeholder="large size" size="lg" />
</div>
```

### Textarea variants

Use the `variant` prop to change the visual style of the Textarea.
You can set the value to `outline`, `flushed`, `filled` or `unstyled`.

<ComponentContainer>
  <Textarea variant="outline" placeholder="outline" />
  <Textarea variant="flushed" placeholder="flushed" />
  <Textarea variant="filled" placeholder="filled" />
  <Textarea variant="unstyled" placeholder="unstyled" />
</ComponentContainer>

```jsx
<div className="flex items-center gap-2">
  <Textarea variant="outline" placeholder="outline" />
  <Textarea variant="flushed" placeholder="flushed" />
  <Textarea variant="filled" placeholder="filled" />
  <Textarea variant="unstyled" placeholder="unstyled" />
</div>
```

### Props

##### `variant`

- Description: The visual style of the textarea.
- Type: `unstyled` | `outline` | `flushed` | `filled`
- Default: `outline`

##### `size`

- Description: The size of the textarea's text.
- Type: `xs` | `sm` | `md` | `lg`
- Default: `md`

##### `isInvalid`

- Description: If `true`, the form control will be invalid. This has 2 side effects: - The FormLabel
  and FormErrorIcon will have `data-invalid`
  set to `true` - The form element (e.g, Input) will have `aria-invalid` set to `true`
- Type: `boolean`
- Default: `false`

##### `isReadOnly`

- Description: If `true`, the form control will be `readonly`
- Type: `boolean`
- Default: `false`

##### `isRequired`

- Description:
  If `true`, the form control will be required. This has 2 side effects: - The FormLabel will show a required indicator - The form
  element (e.g, Input) will have `aria-required` set to `true`
- Type: `boolean`
- Default: `false`
